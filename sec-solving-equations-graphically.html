    Section 14.2 Solving Equations Graphically

    Graphing calculators, like Desmos, or the TI-84 Plus CE, can solve equations graphically very easily. The basic idea is to graph both sides of the equation, and the solutions are the intersection points.
    🔗

    Subsection 14.2.1 Solving Equations Using a Graphing Calculator (Intersections)

    Example 14.2.1. Solving by Intersection.

    Solve the equation \(x^2 = x + 2\text{.}\) To do this, graph both sides of the equation, \(y = x^2\) and \(y = x + 2\) (Desmos graph). Observe the intersection points are at \(x = 2\) and \(x = -1\text{.}\) You can check that these are solutions, by plugging in \(x = 2\) and \(x = -1\) and verifying that both sides are equal,
    🔗
    \begin{align*} x = -1 \quad \rightarrow \quad (-1)^2 \amp = -1 + 2\\ 1 \amp = 1\\ x = 2 \quad \rightarrow \quad 2^2 \amp = 2 + 2\\ 4 \amp = 4 \end{align*}
    Intuitively, this is because the equation \(x^2 = x + 2\) is asking,
    🔗
    “For which values of \(x\) do the two expressions give the same number?”
    🔗
    Then, a point is an intersection if it lies on both graphs, meaning that for that particular \(x\)-value, the \(y\) value for the left side is the same as the \(y\)-value for the right side. That’s what it means for the two sides of the equation to be equal.
    🔗
    🔗
    In general, we want to solve an equation of the form,
    🔗
    \begin{gather*} f(x) = g(x) \end{gather*}
    where \(f, g\) are expressions involving \(x\) (that represent the two sides of the equation).
    🔗
    1. Graph both \(y = f(x)\) and \(y = g(x)\text{.}\)
      🔗
      🔗
    2. The solutions are the \(x\)-values of any intersection points of the two graphs.
      🔗
      🔗
    In short,
    🔗
    \begin{gather*} \boxed{\begin{pmatrix} \text{solutions to} \\ f(x) = g(x) \end{pmatrix}     \Longleftrightarrow     \begin{pmatrix} \text{intersections of} \\ y = f(x) \text{ and } y = g(x) \end{pmatrix}} \end{gather*}
    🔗

    Subsection 14.2.2 Solving Numerically by Finding Zeros (Reducing to One Graph)

    In fact, there is a slightly easier way. The general problem of solving an equation of the form,
    🔗
    \begin{gather*} f(x) = g(x) \end{gather*}
    can be rewritten by moving everything to one side (say, subtracting \(g(x)\) from both sides), to get \(f(x) - g(x) = 0\text{.}\) Then, the equation is in the form of one function equal to 0,
    🔗
    \begin{gather*} F(x) = 0 \end{gather*}
    where \(F(x) = f(x) - g(x)\text{.}\) This transforms the problem from solving an equation with two separate sides, to finding the root of a (single) function. Then, to solve,
    🔗
    1. Graph \(y = f(x) - g(x)\text{.}\)
      🔗
      🔗
    2. The solutions are the \(x\)-intercepts (zeros) of the graph.
      🔗
      🔗
    In other words,
    🔗
    \begin{gather*} \boxed{\begin{pmatrix} \text{solutions to} \\ \text{equation} = 0 \end{pmatrix}     \Longleftrightarrow     \begin{pmatrix} \text{zeros of the} \\ \text{graph} \end{pmatrix}} \end{gather*}
    🔗

    Subsection 14.2.3 Comparing the Two Methods

    Both the intersection method and the zeros method are mathematically equivalent, in that both will give you the same answer. However, there are some advantages and disadvantages to each, both practically and conceptually.
    🔗
    • The intersection method is more intuitive, because you can visually see where the two functions intersect. If the functions represent some quantity, then it also has a useful interpretation.
      🔗

      Example 14.2.2. Interpretating Intersection.

      For example, if \(y\) represents how far each object has gone in \(x\) seconds, then the intersection point \((x,y)\) means that after \(x\) seconds, both objects have travelled the same distance \(y\text{.}\) In contrast, with the zeros method, the \(y\)-coordinate is always 0 and has no interpretation.
      🔗
      🔗
      🔗
    • The zeros method is more computationally simpler, because you only have one function. Graphically, you only have to look near the \(x\)-axis to find solutions. This is especially helpful when the functions are huge, and so you’d have to zoom out a lot to see the intersections.
      🔗

      Example 14.2.3. Solving with the Zeros Method.

      Solve the equation \(\frac{1}{5}x^2 = 5x + 2\text{.}\) Using the intersection method, there is 1 intersection visible, but there is another intersection that is only visible if you zoom out a lot. Instead, using the zeros method, move all terms to one side, to get \(\frac{1}{5}x^2 - 5x - 2 = 0\text{.}\) Then, graphing this function, you can more easily see the 2 zeros of the graph, which are the solutions.
      🔗
      🔗
      🔗
    🔗

    Subsection 14.2.4 Solving Equations Graphically on the TI-84 Plus CE

    With the TI-84 Plus CE, the strategy is similar to using Desmos, except it requires a few more button presses. Again, you can either do the intersection method (\(f(x) = g(x)\)) or the zeros method (\(f(x) = 0\)).
    🔗
    1. Enter the functions: Press [Y=] to to to the function editor. Enter the left side of the equation in Y1 and the right side in Y2. For example, for the equation \(2x + 3 = 5x - 1\text{,}\) put \(2x + 3\) in Y1 and \(5x - 1\) in Y2. If solving \(f(x) = 0\text{,}\) enter the expression in Y1 (and leave Y2 blank).
      🔗
      🔗
    2. Adjust the window (if needed): Press [WINDOW] to set appropriate viewing bounds for Xmin, Xmax, Ymin, and Ymax so the graph and intersection points are visible. You can use [ZOOM] \(\rightarrow\) 6 (ZStandard) for a default view.
      🔗
      🔗
    3. Graph the functions: Press [GRAPH] to display the graphs.
      🔗
      🔗
    4. Find the intersection or root: Press [2nd] [TRACE] (CALC menu), then select 5: intersect (for intersection of two functions) or 2: zero (for roots of a single function). It will ask you to set a lower bound (Left Bound?) and upper bound (Right Bound?). These bounds define the interval for \(x\) where the calculator will search for the solution, so make sure it’s between them. Move the cursor with the arrow keys, and press [ENTER] for each.
      🔗
      It will also ask for a Guess?, which doesn’t really matter, you can just press [ENTER] again (technically, the guess helps refine the search, but it only really helps for complicated functions that have many roots which are close together).
      🔗
      🔗
